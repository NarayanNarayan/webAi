Build a simple LLM application with chat models and prompt templates

In this quickstart we‚Äôll show you how to build a simple LLM application\nwith LangChain. This application will translate text from English into\nanother language. This is a relatively simple LLM application - it‚Äôs\njust a single LLM call plus some prompting. Still, this is a great way\nto get started with LangChain - a lot of features can be built with just\nsome prompting and an LLM call!

After reading this tutorial, you‚Äôll have a high level overview of:

  • Using language models

  • Using prompt templates

  • Debugging and tracing your application using\nLangSmith

Let‚Äôs dive in!

Setup‚Äã

Installation‚Äã

To install LangChain run:

  • npm
  • yarn
  • pnpm
npm i langchain @langchain/core
yarn add langchain @langchain/core
pnpm add langchain @langchain/core

For more details, see our Installation\nguide.

LangSmith‚Äã

Many of the applications you build with LangChain will contain multiple\nsteps with multiple invocations of LLM calls. As these applications get\nmore and more complex, it becomes crucial to be able to inspect what\nexactly is going on inside your chain or agent. The best way to do this\nis with LangSmith.

After you sign up at the link above, make sure to set your environment\nvariables to start logging traces:

export LANGSMITH_TRACING="true"
export LANGSMITH_API_KEY="..."

# Reduce tracing latency if you are not in a serverless environment
# export LANGCHAIN_CALLBACKS_BACKGROUND=true

Pick your chat model:

  • Groq
  • OpenAI
  • Anthropic
  • Google Gemini

Because chat models are Runnables, they\nexpose a standard interface that includes async and streaming modes of\ninvocation. This allows us to stream individual tokens from a chat\nmodel:

const stream = await model.stream(messages);

const chunks = [];
for await (const chunk of stream) {
  chunks.push(chunk);
  console.log(`${chunk.content}|`);
}

Prompt Templates‚Äã

Right now we are passing a list of messages directly into the language\nmodel. Where does this list of messages come from? Usually, it is\nconstructed from a combination of user input and application logic. This\napplication logic usually takes the raw user input and transforms it\ninto a list of messages ready to pass to the language model. Common\ntransformations include adding a system message or formatting a template\nwith the user input.

Prompt templates are a concept in\nLangChain designed to assist with this transformation. They take in raw\nuser input and return data (a prompt) that is ready to pass into a\nlanguage model.

Let‚Äôs create a prompt template here. It will take in two user variables:

  • language: The language to translate text into
  • text: The text to translate
import { ChatPromptTemplate } from "@langchain/core/prompts";

First, let‚Äôs create a string that we will format to be the system\nmessage:

const systemTemplate = "Translate the following from English into {language}";

Next, we can create the PromptTemplate. This will be a combination of\nthe systemTemplate as well as a simpler template for where to put the\ntext

const promptTemplate = ChatPromptTemplate.fromMessages([
  ["system", systemTemplate],
  ["user", "{text}"],
]);

Note that ChatPromptTemplate supports multiple message\nroles in a single template. We format\nthe language parameter into the system message, and the user text\ninto a user message.

The input to this prompt template is a dictionary. We can play around\nwith this prompt template by itself to see what it does by itself

const promptValue = await promptTemplate.invoke({
  language: "italian",
  text: "hi!",
});
